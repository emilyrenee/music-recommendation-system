Music Recommendation System¶

Problem Definition¶

The Context:¶

Recommendations drive modern music streaming services. Users expect passive discovery, where new songs and fresh artists surface effortlessly. Behind this experience lies a finely tuned system. Popularity and trends shape a fair amount of listening behavior, defining what listeners engage with on a large scale. At its essence, music is a shared cultural experience—a soundtrack to moments that unite us. At the same time, music is also deeply personal. We seek music out as a form of self-reflection and self-understanding. A song that just gets us is always just a click away. We might lean toward a favorite genre, but when our hearts break, we instinctively turn to our favorite love song. When we celebrate, we need an anthem.

Achieving this means leveraging both collaborative filtering (to surface widely liked content) and content-based filtering (to uncover hidden gems tailored to the individual). This project applies these concepts to explore how machine learning can bridge the gap between mass appeal and personal resonance.

The objective:¶

The goal of this project is to build a data-driven music recommendation system that suggests relevant songs based on user interaction data and text-based similarity. By leveraging collaborative filtering (CF) models and content-based filtering, the model should be able to:

  • Predict songs that a user is likely to enjoy by analyzing past play counts and patterns in user behavior.
  • Surface songs based on textual similarity while recognizing the limitations of available metadata (title, artist, release, and year).

The key questions:¶

  • How do users interact with songs?
  • What recommendation approach performs best with the data we have?
  • How do we evaluate recommendation effectiveness?
  • Can we leverage feature engineering to improve recommendations?

The problem formulation:¶

We will use a dataset containing 1,000,000 unique songs and 2,000,000 user-song play interactions to build our system. By analyzing listening behavior, play counts, and song features, the model will predict songs a user might enjoy and suggest them accordingly. The goal is to improve user engagement by presenting relevant and diverse recommendations while handling data challenges such as missing values, duplicate interactions, and data sparsity.

Data Dictionary¶

The core data is the Taste Profile Subset released by the Echo Nest as part of the Million Song Dataset. There are two files in this dataset. The first file contains the details about the song id, titles, release, artist name, and the year of release. The second file contains the user id, song id, and the play count of users.

song_data

  • song_id - A unique id given to every song
  • title - Title of the song
  • Release - Name of the released album
  • Artist_name - Name of the artist
  • year - Year of release

count_data

  • user _id - A unique id given to the user
  • song_id - A unique id given to the song
  • play_count - Number of times the song was played

# Build the function to find top n songs
def get_top_n_songs(df, n=10, metric="play_frequency"):
    """
    Returns the top n songs based on a given metric.

    Parameters:
    - df (DataFrame): The DataFrame containing song stats.
    - n (int): The number of top songs to return (default=10).
    - metric (str): The metric to rank songs by. Options:
        - "play_frequency" (default): Most played songs (number of unique user interactions).
        - "avg_play_count": Songs with the highest average play count per user.

    Returns:
    - DataFrame: Top `n` songs ranked by the specified metric.
    """

    if metric not in ["play_frequency", "avg_play_count"]:
        raise ValueError("Invalid metric. Choose 'play_frequency' or 'avg_play_count'.")

    # Sort by the chosen metric in descending order and return top n
    top_songs = df.sort_values(by=metric, ascending=False).head(n)

    return top_songs

# Dictionary to store model evaluation results incrementally
model_comparison_results = {}

def precision_recall_at_k(model, model_name, testset, k=DEFAULT_K, threshold=DEFAULT_THRESHOLD):
    """
    Evaluates a recommendation model and stores results **without any output**.

    Parameters:
    - model (Surprise Model): The trained model to evaluate.
    - model_name (str): Name of the model for storage & reference.
    - k (int): Number of recommendations to consider.
    - threshold (float): Minimum rating considered as "relevant."

    Returns:
    - Dictionary of model evaluation results (RMSE, Precision@K, Recall@K, F1-score).

    Updates:
    - Stores results in `model_comparison_results` to avoid redundant computation.
    """

    # Skip evaluation if already stored
    if model_name in model_comparison_results:
        return model_comparison_results[model_name]

    # Generate evaluation predictions
    predictions = model.test(testset)
    rmse_value = accuracy.rmse(predictions, verbose=False)  # No print output

    # Map evaluation predictions to each user
    user_est_true = defaultdict(list)
    for uid, _, true_r, est, _ in predictions:
        user_est_true[uid].append((est, true_r))

    precisions, recalls = {}, {}

    for uid, user_ratings in user_est_true.items():
        user_ratings.sort(key=lambda x: x[0], reverse=True)

        # Compute precision and recall
        n_rel = sum(true_r >= threshold for _, true_r in user_ratings)
        n_rec_k = sum(est >= threshold for est, _ in user_ratings[:k])
        n_rel_and_rec_k = sum((true_r >= threshold and est >= threshold) for est, true_r in user_ratings[:k])

        precisions[uid] = n_rel_and_rec_k / n_rec_k if n_rec_k != 0 else 0
        recalls[uid] = n_rel_and_rec_k / n_rel if n_rel != 0 else 0

    # Compute averages
    precision_avg = round(sum(precisions.values()) / len(precisions), 3)
    recall_avg = round(sum(recalls.values()) / len(recalls), 3)
    f1_score = round((2 * precision_avg * recall_avg) / (precision_avg + recall_avg), 3) if (precision_avg + recall_avg) > 0 else 0

    # Store results
    model_comparison_results[model_name] = {
        "RMSE": rmse_value,
        "Precision@k": precision_avg,
        "Recall@k": recall_avg,
        "F1-score": f1_score
    }

    return model_comparison_results[model_name]

def make_and_store_prediction(model, model_name, user_id, song_id, actual_play_count=None):
    """
    Predicts the play count for a given user and song using the provided model.

        'Song ID': song_id,
        'Predicted Play Count': round(pred.est, 2),

        'Prediction Error': round(prediction_error, 2) if prediction_error != "N/A" else "N/A"
    }
    predictions_store[user_id].append(entry)

def get_recommendations(df, user_id, model, top_n=5):
    """
    Generates top_n song recommendations for a given user using a specified recommendation model.

    Parameters:
    - user_id (str/int): The user ID for whom recommendations are generated.
    - model: The trained recommendation model (Co-Clustering, SVD, User-User, Item-Item, etc.).
    - df (DataFrame): Dataset containing song metadata (song_id, title, artist_name, release).
    - top_n (int): Number of recommendations to return (default=5).

    Returns:
    - DataFrame with top_n recommended songs and estimated play counts.
    """

    # Get all songs the user has interacted with
    heard_songs = set(df[df["user_id"] == user_id]["song_id"].unique())

    # Get all unique song IDs in dataset
    all_song_ids = set(df["song_id"].unique())

    # Find songs the user has NOT played
    unseen_songs = list(all_song_ids - heard_songs)

    # Predict play count for all unseen songs
    predictions = [model.predict(user_id, song_id) for song_id in unseen_songs]

    # Sort predictions by estimated play count (descending order) and select top N
    top_predictions = sorted(predictions, key=lambda x: x.est, reverse=True)[:top_n]

    # Extract song IDs from top predictions
    top_song_ids = [pred.iid for pred in top_predictions]

    # Retrieve song details from the original dataframe
    top_songs = df[df["song_id"].isin(top_song_ids)][["song_id", "title", "artist_name", "release", "year"]].drop_duplicates()

    # Merge with predictions to include estimated play counts
    top_songs = top_songs.merge(
        pd.DataFrame([(pred.iid, pred.est) for pred in top_predictions], columns=["song_id", "predicted_play_count"]),
        on="song_id",
        how="left"
    )

    return top_songs

def ranking_songs(recommendations, playing_count):
    """
    Adjusts and ranks recommended songs based on corrected play counts.

    Parameters:
    - recommendations: DataFrame with 'song_id' and 'predicted_play_count'.
    - playing_count: DataFrame with 'song_id' and 'play_count' (actual play count).

    Returns:
    - DataFrame with ranked songs and corrected play counts.
    """

    # Ensure both song_id columns are of type string for a reliable merge
    recommendations["song_id"] = recommendations["song_id"].astype(str)
    playing_count["song_id"] = playing_count["song_id"].astype(str)

    # Merge actual play counts with recommendations
    ranked_songs = recommendations.merge(playing_count, on="song_id", how="left")

    # Replace missing play counts with 1 (for unseen songs)
    ranked_songs["play_count"] = ranked_songs["play_count"].fillna(1)

    # Apply correction: predicted_play_count - (1 / sqrt(play_count))
    ranked_songs["corrected_play_count"] = (
        ranked_songs["predicted_play_count"] - (1 / np.sqrt(ranked_songs["play_count"]))
    )

    # Rank songs based on corrected play counts (higher is better)
    ranked_songs = ranked_songs.sort_values(by="corrected_play_count", ascending=False).reset_index(drop=True)

    return ranked_songs

In clustering-based recommendation systems, we explore the similarities and differences in people's tastes in songs based on how they rate different songs. We cluster similar users together and recommend songs to a user based on play_counts from other users in the same cluster.

def recommendations(title, similar_songs):
    """
    Function that takes in a song title and returns the top 10 recommended songs
    based on content similarity, while avoiding recommendations where the song title
    directly matches or significantly overlaps with the artist name.

    Parameters:
    - title (str): The title of the input song.
    - similar_songs (numpy.ndarray): The similarity matrix (cosine similarity).

    Returns:
    - DataFrame: Top 10 recommended songs with their title, artist name, and album (release).
    """
    # Ensure the song exists in the index
    if title not in indices:
        return pd.DataFrame({"Error": ["Song not found in the dataset."]})

    # Step 1: Get the index of the song that matches the title
    idx = indices[title]
    if isinstance(idx, pd.Series):
        idx = idx.iloc[0]  # Handle duplicate song titles

    # Step 2: Get similarity scores for this song and sort them in descending order
    sim_scores = list(enumerate(similar_songs[idx]))
    sim_scores = sorted(sim_scores, key=lambda x: x[1], reverse=True)

    # Step 3: Retrieve the song details (title, artist, release)
    song_indices = [i[0] for i in sim_scores]
    recommended_songs = df_small_content.iloc[song_indices][["title", "artist_name", "release"]]

    # Step 4: Filter out songs where the title appears in the artist's name
    recommended_songs = recommended_songs[
        ~recommended_songs.apply(lambda row: row["title"].lower() in row["artist_name"].lower(), axis=1)
    ]

    # Step 5: Keep only the top 10 recommendations
    return recommended_songs.head(10).reset_index(drop=True)  # Reset index for clean display

def evaluate_content_based(model, df, test_users, k=10, threshold=1.8):
    """
    Evaluates the content-based recommendation model using Precision@k, Recall@k, and F1-score.

    Parameters:
    - model: The trained similarity matrix (cosine similarity).
    - df: The dataset with song metadata.
    - test_users: A list of user IDs for evaluation.
    - k: Number of recommendations to consider.
    - threshold: Play count threshold for relevance.

    Returns:
    - Dictionary with Precision@k, Recall@k, and F1-score.
    """
    precisions, recalls = [], []

    for user in test_users:
        # Get songs the user has interacted with
        user_songs = df[df["user_id"] == user]["title"].tolist()

        # Get recommendations for each song the user played
        recommended_songs = []
        for song in user_songs:
            recommended_songs.extend(recommendations(song, model))

        # Keep only top-k recommendations
        recommended_songs = list(set(recommended_songs))[:k]
        relevant_songs = set(user_songs)

        # Compute number of correctly recommended items
        num_correct = sum(1 for song in recommended_songs if any(song in s for s in relevant_songs))

        # Compute precision and recall
        precision = num_correct / len(recommended_songs) if recommended_songs else 0
        recall = num_correct / len(relevant_songs) if relevant_songs else 0

        precisions.append(precision)
        recalls.append(recall)

    # Compute mean Precision, Recall, and F1-score
    mean_precision = np.mean(precisions)
    mean_recall = np.mean(recalls)
    f1_score = (
        2 * (mean_precision * mean_recall) / (mean_precision + mean_recall)
        if (mean_precision + mean_recall) > 0 else 0
    )

    return {"Precision@k": round(mean_precision, 3), "Recall@k": round(mean_recall, 3), "F1-score": round(f1_score, 3)}

Observations on Content-Based Recommendations:

Note: The content-based model generates recommendations based solely on song similarity, without considering user listening history. This makes it well-suited for an "Up Next" queue where continuity in playback is prioritized. However, for personalized discovery, filtering out previously played songs or incorporating user preferences would be necessary.

  • TF-IDF Overemphasizes Exact Matches

    • The TF-IDF model tends to recommend songs from the same artist and album as the input song.
    • For Learn To Fly and At The Bottom Of Everything, nearly all recommendations come from the same album.
    • Similarly named albums and songs may introduce irrelevant recommendations, such as Disturbed’s Down With the Sickness being suggested for a song from Bright Eyes’ Motion Sickness album
